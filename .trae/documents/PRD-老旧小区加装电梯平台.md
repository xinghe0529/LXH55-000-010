## 1. 产品概述

老旧小区加装电梯居民投票与工程进度追踪平台，解决老旧小区电梯加装过程中居民意见征集不透明、费用分摊争议大、工程进度信息不对称等痛点。
- 目标用户：小区居民（业主）、业委会/物业管理员、工程施工方
- 核心价值：数字化投票流程、透明化费用分摊、可视化进度追踪、异议申诉机制

## 2. 核心功能

### 2.1 用户角色
| 角色 | 注册方式 | 核心权限 |
|------|---------|---------|
| 居民业主 | 房号绑定注册 | 浏览提案、参与投票、查看分摊费用、提交申诉、查看进度 |
| 业委会管理员 | 后台授权 | 发起/关闭提案、公示结果、录入进度、管理资金明细 |
| 施工方 | 后台授权 | 更新施工进度节点、上传工程报告 |

### 2.2 功能模块
1. **提案广场**：展示所有加装电梯提案卡片、状态筛选、提案详情、发起新提案
2. **投票面板**：投票选项（同意/反对/弃权）、楼层系数说明、费用分摊估算表、投票结果公示、异议申诉入口
3. **进度看板**：施工时间轴、里程碑节点、资金使用明细表格、工程文档下载

## 3. 核心流程

**主要用户流程描述：**
1. 业委会管理员在提案广场发起某楼栋加装电梯提案，填写楼栋基本信息和工程预估总价
2. 系统根据楼层系数算法自动计算每户分摊金额，生成费用估算表
3. 居民进入投票面板，选择自己的房号，查看分摊费用后进行投票（同意/反对/弃权）
4. 投票期结束后系统自动统计结果并公示，投票权重按楼层系数加权计算
5. 若居民对投票过程有异议，可在公示期内提交申诉
6. 投票通过后提案进入施工阶段，管理员在进度看板更新施工节点和资金支出
7. 居民随时查看进度时间轴和资金使用明细，直至工程完工验收

```mermaid
    H -->|是| I["启动工程施工"]

    I --> K["更新进度+资金明细"]
    K --> L["居民查看进度看板"]
    L --> M["工程完工验收"]
```

## 4. 用户界面设计

### 4.1 设计风格
- **主色调**：深青蓝色 `#0F4C5C`（专业、信任感），辅色：暖橙色 `#E36414`（行动号召）
- **中性色**：石板灰系列（`#F8FAFC` `#E2E8F0` `#64748B` `#1E293B`）
- **按钮风格**：圆角矩形（8px），主按钮实色填充+微阴影，次按钮描边样式
- **字体**：中文使用 Noto Serif SC（标题，稳重有质感）+ PingFang SC（正文，现代清晰）
- **布局风格**：顶部导航栏 + 三栏式卡片布局，带细微噪点纹理背景
- **图标风格**：Lucide 线性图标，24px 标准尺寸

### 4.2 页面设计概述
| 页面名称 | 模块名称 | UI元素 |
|---------|---------|---------|
| 提案广场 | 顶部Hero | 大标题+统计数据面板（提案总数/施工中/已完工）+ 渐变背景带建筑剪影 |
| 提案广场 | 筛选栏 | 状态Tab按钮组 + 搜索框 + 发起提案主按钮 |
| 提案广场 | 提案卡片 | 左：楼栋号大字+楼层标签，中：状态徽章+进度条+描述，右：操作按钮组 |
| 投票面板 | 身份区 | 楼栋/单元/楼层/房号四级下拉选择器 |
| 投票面板 | 费用表 | 斑马纹表格，当前用户行高亮，合计行加粗 |
| 投票面板 | 投票区 | 三个大尺寸选项卡（同意绿/反对红/弃权灰），选中态有边框+对勾动画 |
| 投票面板 | 结果区 | 三色堆叠进度条 + 百分比数字 + 权重说明提示 |
| 进度看板 | 时间轴 | 左侧垂直色带+节点圆点，右侧卡片含标题/日期/负责人/描述/附件 |
| 进度看板 | 资金表 | 分类汇总标签页 + 明细行带展开箭头 + 预算vs实际对比条形图 |

### 4.3 响应式
- **桌面优先**：1440px基准设计，使用Grid布局
- **平板适配**：1024px断点，三栏变两栏，卡片间距紧凑
- **移动端**：768px断点，单栏堆叠，底部Tab导航替代顶部菜单
- **触控优化**：按钮最小高度44px，表格支持横向滚动
